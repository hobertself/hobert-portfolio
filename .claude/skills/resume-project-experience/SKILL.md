# 项目经历转化 Skill

## 功能概述
将任何形式的项目经历转化为高质量的简历项目描述，使用STAR法则和四字词语，突出个人价值和成果。

## 核心功能
1. **STAR法则结构化**
   - 背景（Situation）：项目背景和任务
   - 职责（Task）：具体职责和目标
   - 行动（Action）：采取的行动，动词开头
   - 成果（Result）：量化成果和影响

2. **四字词语概括**
   - 统筹规划、需求分析、系统设计、核心开发
   - 性能优化、技术攻关、团队协作、项目管理
   - 效率提升、成本控制、质量保证、用户增长

3. **量化成果展示**
   - 提升数据：性能提升、用户增长
   - 优化成果：流程优化、架构优化
   - 节省成本：时间节省、成本节约
   - 获得认可：奖项、满意度

## 输入处理
支持多种输入形式：
- **项目文档**：提供文件路径（PDF/Word）
- **文字描述**：直接粘贴项目内容
- **要点列表**：分点描述工作内容
- **对话交流**：通过问答收集信息

## 工作流程
1. **接收输入**：确认用户提供的信息形式
2. **内容整理**：提取关键项目信息
3. **STAR构建**：按STAR法则组织内容
4. **词语优化**：匹配合适的四字词语
5. **量化提炼**：提取和量化成果数据
6. **生成输出**：完成简历项目描述

## 输出格式 - STAR法则

按以下结构撰写项目经历，每点必须用**四字词语**概括通用能力：

```
【四字词语】项目背景和任务描述

【四字词语】具体职责和目标说明

【四字词语】实际采取的行动陈述（多用动词开头）

【四字词语】量化成果和影响强调
```

### 四字词语示例
- 统筹规划、需求分析、系统设计、核心开发
- 性能优化、技术攻关、团队协作、项目管理
- 效率提升、成本控制、质量保证、用户增长
- 数据分析、问题解决、创新突破、价值创造

### 量化成果要求
尽可能用数据体现成果：
- 提升/降低：**提升30%转化率**、**降低50%响应时间**
- 优化：**优化用户注册流程**、**重构核心代码架构**
- 节省：**节省20%开发时间**、**节省10万元成本**
- 获得：**获得团队最佳实践奖**、**获得用户好评率98%**

## 撰写原则
1. **信息准确**：基于原文档内容，不捏造信息
2. **字数凝练**：每点控制在 30-60 字，总字数不超过 300 字
3. **动词开头**：行动描述用"设计、开发、优化、推进"等动词
4. **逻辑清晰**：STAR 四个层次分明，层层递进
5. **通用能力**：提炼出可迁移的通用能力，而非具体技术细节

## 触发场景
当用户出现以下表述时自动触发：
- "帮我写项目经历"
- "这个项目怎么写进简历"
- "把我的项目经历改改"
- "简历项目经历怎么写"
- "帮我优化项目描述"
- "项目经历怎么写更专业"

## 使用指南

### 情景1：有项目文档
```
用户：我有一个项目文档，需要转化为简历项目经历
响应：请提供文档路径，我将读取并分析内容
```

### 情景2：有项目描述
```
用户：这是我的项目描述...
响应：我将基于你的描述进行优化和结构化
```

### 情景3：需要引导
```
用户：我不知道怎么写项目经历
响应：让我通过几个问题帮你梳理项目信息
```

## 示例

```
【统筹规划】负责电商平台用户中心模块的架构设计与功能迭代

【需求分析】深入理解业务需求，制定详细的技术方案和开发计划

【核心开发】主导核心代码开发，优化数据库查询效率，完善异常处理机制

【效率提升】将接口响应时间从 500ms 优化至 150ms，用户留存率提升 15%
```

## 质量控制
- 确保STAR结构完整
- 四字词语准确概括
- 量化数据真实有效
- 语言简洁专业
- 突出个人价值贡献
