# 项目经历转化 Skill

将项目经历转化为简历项目描述。使用STAR法则结构化项目经历，用四字词语概括能力，量化成果展示。当用户提到"简历"、"项目经历"、"项目描述"时使用。

## 功能概述
将任何形式的项目经历转化为高质量的简历项目描述，使用STAR法则和四字词语，突出个人价值和成果。

## 核心功能
1. **STAR法则结构化**
   - 背景（Situation）：项目背景和任务
   - 职责（Task）：具体职责和目标
   - 行动（Action）：采取的行动，动词开头
   - 成果（Result）：量化成果和影响

2. **四字词语概括**
   - 统筹规划、需求分析、系统设计、核心开发
   - 性能优化、技术攻关、团队协作、项目管理
   - 效率提升、成本控制、质量保证、用户增长

3. **量化成果展示**
   - 提升数据：性能提升、用户增长
   - 优化成果：流程优化、架构优化
   - 节省成本：时间节省、成本节约
   - 获得认可：奖项、满意度

## 触发场景
当用户出现以下表述时自动触发：
- "帮我写项目经历"
- "这个项目怎么写进简历"
- "把我的项目经历改改"
- "简历项目经历怎么写"
- "帮我优化项目描述"
- "项目经历怎么写更专业"

## 使用指南

### 情景1：有项目文档
```
用户：我有一个项目文档，需要转化为简历项目经历
响应：请提供文档路径，我将读取并分析内容
```

### 情景2：有项目描述
```
用户：这是我的项目描述...
响应：我将基于你的描述进行优化和结构化
```

### 情景3：需要引导
```
用户：我不知道怎么写项目经历
响应：让我通过几个问题帮你梳理项目信息
```

## 输入支持
- 项目文档（PDF/Word）
- 文字描述
- 要点列表
- 对话交流
