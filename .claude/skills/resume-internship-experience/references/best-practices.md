# 实习经历转化最佳实践

## 命名规范

- Skill名称使用小写字母+数字+连字符，如：`resume-project-experience`
- 避免模糊名称：`helper`, `utils`, `tools`
- 推荐动名词形式：`processing-pdfs`, `analyzing-spreadsheets`

## Description编写指南

- **始终用第三人称**: "处理Excel文件" ✅ / "我帮你处理" ❌
- **包含触发场景**: "当用户提到PDF、表单时使用"
- **清晰描述功能**: 既要说明做什么，也要说明何时使用

## STAR法则应用要点

### Situation (背景)
- 简明扼要说明工作背景
- 包含必要的上下文信息
- 突出挑战和机遇

### Task (职责)
- 明确具体的工作目标
- 体现职位的核心职责
- 避免过于宽泛的描述

### Action (行动)
- 使用强有力的动词开头
- 描述具体的行动步骤
- 体现个人能力和方法

### Result (成果)
- 优先使用数据量化
- 说明产生的影响和价值
- 展示个人贡献和能力

## 量化表达模式

### 百分比提升
- "提升了XX%的效率"
- "降低了XX%的成本"
- "增加了XX%的转化率"

### 具体数字
- "处理了XX+个项目"
- "管理XX人团队"
- "优化了XX个流程"

### 时间节省
- "节省了XX小时/天"
- "将周期从X缩短到Y"
- "提前XX天完成"

### 质量提升
- "获得XX%好评率"
- "减少XX%错误率"
- "达到XX%准确率"

## 行业术语优化建议

### 互联网/科技行业
- 避免口语化：用"用户体验"而非"用户感受"
- 专业术语：用"敏捷开发"而非"快速迭代"
- 技术栈：明确列出使用的技术和工具

### 金融行业
- 风险管理：体现风控意识和专业判断
- 数据分析：突出量化能力和专业工具
- 合规意识：强调合规性和专业性

### 咨询行业
- 解决方案：体现系统思考和解决方案设计
- 客户导向：强调客户价值和满意度
- 项目管理：体现项目管理能力和交付能力

## 能力维度映射

### 核心能力框架
1. **专业技能**: 行业知识和技能
2. **解决问题**: 分析和解决问题的能力
3. **沟通协作**: 团队合作和沟通能力
4. **学习能力**: 快速学习和适应能力
5. **执行力**: 任务执行和交付能力

### 映射方法
- 根据工作内容识别能力类型
- 用具体事例支撑能力描述
- 保持能力描述的一致性
